# 系统规范目录 (System Specifications)

## 目录说明
此目录包含已上线功能的完整、最新规范。所有规范文件都是系统的**单一事实源**。

## 规范状态
- **状态**: 只读 (Read-Only)
- **更新方式**: 仅可通过归档流程更新
- **权威性**: 所有系统行为以此目录内容为准

## 规范文件结构
```
specs/
├── <module-a>/          # 模块 A 规范目录
│   └── spec.md          # 模块 A 详细行为规则
├── <module-b>/          # 模块 B 规范目录
│   └── spec.md          # 模块 B 详细行为规则
└── ...
```

## 使用规则
1. 开发新功能前必须先阅读相关模块规范
2. 测试用例必须基于此目录的规范编写
3. 代码实现必须严格遵循规范定义
4. 发现规范与代码不一致时，优先修改代码而非规范

## 维护责任
- **模块负责人**: 负责维护对应模块的规范准确性
- **技术负责人**: 负责整体规范架构的一致性
- **所有开发者**: 有责任报告发现的规范漂移问题

> **重要**: 此目录内容为系统真相，任何修改都必须通过正式的变更流程。
