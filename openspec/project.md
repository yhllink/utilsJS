## 架构模式 (Architecture)
本项目采用**模块化工具库架构**，每个功能模块独立封装，具有以下特点：

### 模块化设计
- **独立模块**: 每个功能模块独占一个目录，实现单一职责原则
- **无状态设计**: 大多数函数为纯函数，不依赖外部状态
- **最小依赖**: 仅依赖 lossless-json 用于 JSON 序列化处理

### 构建架构
- **多格式输出**: 同时支持 CommonJS (lib/) 和 ES Modules (es/) 格式
- **Tree Shaking**: 支持按需导入，优化打包体积
- **类型声明**: 自动生成 TypeScript 声明文件

### 测试架构
- **模块化测试**: 每个功能模块对应独立的测试文件
- **环境模拟**: 配置 jsdom 环境支持浏览器 API 测试
- **覆盖率监控**: 集成覆盖率报告，确保代码质量

## 目录结构约定 (Directory Conventions)

### 源代码结构
```
src/
├── <ModuleName>/           # 功能模块目录，使用 PascalCase
│   └── <ModuleName>.ts     # 模块入口文件，与目录同名
├── vendors/                # 第三方依赖封装
│   └── lossless-json.ts    # JSON 处理工具封装
└── index.ts               # 主入口文件，自动生成导出
```

### 测试结构
```
test/
└── unit/                   # 单元测试目录
    └── <ModuleName>/       # 对应模块测试目录
        └── <ModuleName>.test.ts  # 测试文件，与源文件同名
```

### 构建输出
```
lib/                        # CommonJS 格式输出
es/                         # ES Modules 格式输出
coverage/                   # 测试覆盖率报告
```

## 编码风格 (Coding Style)

### 命名规范
- **函数命名**: camelCase (如: `deepClone`, `quickSort`)
- **类名命名**: PascalCase (如: `PubSub`, `IntersectionObserver`)
- **类型命名**: PascalCase 后缀 Type (如: `ClassNameListType`)
- **常量命名**: UPPER_SNAKE_CASE (如: `MAX_RETRY_COUNT`)

### 类型注解
- **严格模式**: 启用 TypeScript 严格类型检查
- **显式类型**: 避免隐式 any，所有参数和返回值必须显式声明类型
- **接口优先**: 优先使用接口定义复杂数据结构

### 注释规范
- **JSDoc 标准**: 所有导出函数必须包含完整的 JSDoc 注释
- **示例代码**: 每个函数必须包含 @example 使用示例
- **参数说明**: 使用 @param 详细说明参数类型和用途

### 错误处理
- **异常捕获**: 异步操作必须使用 try-catch 或 Promise.catch
- **错误类型**: 使用自定义错误类型区分不同错误场景
- **边界检查**: 所有输入参数必须进行边界条件验证

### 性能优化
- **循环优化**: 避免在循环中创建函数或对象
- **引用处理**: 使用 WeakMap 处理循环引用场景
- **缓存策略**: 合理使用缓存优化重复计算

## 特殊约定 (Special Conventions)

### 模块开发流程
1. **创建模块目录**: 在 src/ 下创建以模块名命名的目录
2. **实现核心功能**: 编写模块主文件，包含完整 JSDoc 注释
3. **编写单元测试**: 在 test/unit/ 对应目录创建测试文件
4. **验证功能正确**: 运行测试确保功能符合预期
5. **更新入口文件**: 自动生成 src/index.ts 导出语句

### 代码质量要求
- **测试覆盖率**: 每个模块必须达到 80% 以上的测试覆盖率
- **类型安全**: 禁止使用 any 类型，必须显式声明类型
- **文档完整**: 所有导出函数必须包含完整的使用示例
- **性能优化**: 避免性能瓶颈，合理使用缓存和优化策略

### 浏览器兼容性
- **现代浏览器**: 支持 Chrome 60+, Firefox 55+, Safari 12+, Edge 79+
- **ES2020 特性**: 使用现代 JavaScript 特性，通过 Babel 转译兼容
- **模块化支持**: 同时支持 CommonJS 和 ES Modules 导入方式

### 安全规范
- **输入验证**: 所有用户输入必须进行验证和转义
- **XSS 防护**: 使用安全的 DOM 操作方式，避免 XSS 攻击
- **数据安全**: 敏感数据处理使用安全的编码方式
